# BG Hub Backend

NestJS API for BG Hub.

## API Docs

- **OpenAPI spec (JSON):** [http://localhost:3000/openapi.json](http://localhost:3000/openapi.json) (served at runtime when the server is running)
- **Scalar UI:** [http://localhost:3000/docs](http://localhost:3000/docs)

A static `openapi.json` is also generated by `npm run openapi:generate` (no server needed). If `/docs` or `/openapi.json` fails when the server runs (e.g. after adding new DTOs), ensure the server starts successfully (run migrations if needed) and that feed/response DTOs are registered in Swagger `extraModels` in `main.ts` and `scripts/generate-openapi.ts`.

## Local dev

```bash
npm run start:dev
```

Set `DATABASE_*` and other options in `.env` (see `.env.example`).

## Feed (MVP)

Feed is available at `GET /me/feed` (requires JWT). It returns follow events, list activity (wishlist/collection), and play log activity, newest first, with cursor-based pagination.

### Migration

Run the feed_events migration:

```bash
npm run migration:run
```

This creates the `feed_events` table and enum type `feed_event_type_enum`.

### Curl examples

**Get feed (default limit 20):**

```bash
curl -s -H "Authorization: Bearer YOUR_JWT" "http://localhost:3000/me/feed"
```

**With limit and cursor:**

```bash
curl -s -H "Authorization: Bearer YOUR_JWT" "http://localhost:3000/me/feed?limit=10"
curl -s -H "Authorization: Bearer YOUR_JWT" "http://localhost:3000/me/feed?limit=10&cursor=2025-03-04T12:00:00.000Z:123e4567-e89b-12d3-a456-426614174000"
```

Response shape: `{ "items": [ { "id", "type", "createdAt", "actor", "targetUser?", "game?", "playLogId?", "text" } ], "nextCursor": "..." | null }`.
